Digital Sanctuary — UI/디자인 시스템 & 로드맵

제공된 "Digital Sanctuary" 고충실도 청사진(3화면)을 코드베이스에 정렬한 마스터 설계. 전체 비전·정보구조·디자인 시스템·데이터 모델을 정의하고, 구현을 S1~S6 슬라이스로 쪼갠다. 각 슬라이스는 진입 시점에 개별 상세 스펙을 따로 쓴다.

1. 배경 & 목표

현재 위키 앱은 동작한다 — 매직링크 로그인, 사용자별 문서 소유(멀티테넌트), BlockNote 에디터, source export, Publish(별도 스펙)까지. 그러나 UI는 디자인 시스템이 없다: 컴포넌트마다 인라인 style, 전역 스타일·토큰·공통 셸 부재, 폰트는 Pretendard 하나. 이번 작업의 목표는 제공된 청사진을 출발점으로 전체 UI/디자인 시스템을 처음 세우고, 청사진이 그리는 위키 워크스페이스를 코어 기능 우선으로 단계 구현하는 것이다.

2. 청사진의 위상 (중요)

전달된 Digital Sanctuary.dc.html + tokens.css + base.css + 스크린샷은 HTML로 만든 디자인 레퍼런스(고충실도 프로토타입)다. 그대로 복사해 쓰는 프로덕션 코드가 아니다.

  • 재구현 대상: 우리 환경(Next.js App Router + Tailwind + BlockNote + lucide-react)에서 이 디자인을 픽셀에 가깝게 재구현한다.
  • 차용: 색·타이포·간격·radius·그림자 등 디자인 토큰, 레이아웃, 컴포넌트 패턴, 아이콘 목록.
  • 재현하지 않음: 프로토타입의 가짜 브라우저 크롬(신호등 점·주소창), 캔버스 배치, "Wiki Dashboard with Tree Navigation" 같은 프레임 라벨 — 모두 전시용 장식.
  • 샘플 콘텐츠(Dr. Julian Vance, 631 pages, 4 contributors, 특정 문서 제목)는 더미. 실제 데이터로 대체.

3. 기반 — 이미 구현된 것 (신규 슬라이스 불필요)

아래는 이미 동작하므로 로드맵에 별도 슬라이스로 넣지 않는다. UI 슬라이스가 이 위에 얹힌다.

7. 화면별 설계 (청사진 요약)

7.1 홈 대시보드 / (Frame 1)

  • 페이지 헤더: h1 "Home" + 메타("Your knowledge workspace · N pages") + 우측 + New Page.
  • 탭: Recently edited(활성·2px 밑줄) · Starred · Created by me · All pages.
  • JUMP BACK IN: 4열 카드(아이콘 타일 + 제목 + "폴더 · Edited N ago").
  • 2단 그리드 1fr 290px:
    • 좌: "Recently edited" 카드 테이블 — 컬럼 NAME / TAG / UPDATED, 각 행에 문서 아이콘·제목·브레드크럼·태그 칩·상대시각·아바타.
    • 우 레일: Starred · Recent activity(점 불릿 피드) · Quick access(Global Graph/Reading Queue/Canvas Mode/Collections → 백로그, 비활성).
  • 푸터: 카피라이트 + Privacy/Terms/API(링크 placeholder).

7.2 읽기 / 아티클 /p/[slug] (Frame 2)

  • flex(48px gap, max 880): 아티클 컬럼 + TOC 사이드(172px, sticky).
  • 아티클: 브레드크럼 → h1(38 serif) → 데크(좌 3px 보더 인용) → h2/본문(serif 15/1.65) → figure(이미지+캡션) → 불릿 → 코드블록(다크) → Editor's Note 콜아웃 → 태그 칩 → 저자·읽기시간.
  • TOC: "ON THIS PAGE"(스크롤 스파이 활성) + "RELATED PAGES".
  • 푸터: "Last synced …" + Privacy/Terms + "N pages indexed".
  • 본문 HTML은 blocksToHtml → sanitize(공개 노출, Publish 스펙 §8). 저자·읽기시간·Related는 S5에서 데이터/계산.

7.3 에디터 /edit/[id] (Frame 3) — BlockNote 유지 + 청사진 룩

  • BlockNote 블록 에디터는 그대로. 청사진의 raw 마크다운 소스 표시는 디자인 표현일 뿐 — 우리는 셸/헤더/툴바 디자인만 채택한다.
  • 에디터 헤더(h56): file-pen-line + 문서 제목(serif 17) + 우측 Share(아웃라인) · Publish(다크, send). Publish는 Publish 스펙의 setDocumentPublished 연결.
  • 툴바(h48): bold/italic/strikethrough | h1/h2 | list/list-ordered | link/image/code/quote · 우측 Preview(eye) 토글. 이 버튼들은 BlockNote의 포맷 명령에 바인딩(BlockNote 기본 툴바를 청사진 룩의 커스텀 툴바로 대체/스타일).
  • 본문: BlockNote 에디터 영역(흰 배경, 넉넉한 패딩). 기존 자동저장 유지.
