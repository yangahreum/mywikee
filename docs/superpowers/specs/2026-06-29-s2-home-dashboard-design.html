S2 — 홈 대시보드 (청사진 Frame 1)

로그인 직후 보는 워크스페이스 홈(/)을 청사진 Frame 1과 최대한 동일하게 구현한다. 페이지 헤더 · 탭 · Jump back in 카드 · Recently edited 테이블 · 우측 레일(Starred / Recent activity / Quick access). 실데이터로 채울 수 있는 곳은 채우고, 아직 데이터가 없는 곳(태그·즐겨찾기)은 청사진 레이아웃을 유지한 채 placeholder/빈 상태로 둔다.

청사진 Frame 1AppShell(S1)실데이터+placeholdervitest

1. 배경 & 범위

S1에서 공통 셸(AppShell)과 디자인 토큰이 섰다. S2는 셸 안의 콘텐츠를 청사진 Frame 1으로 채운다. 현재 /me의 "내 문서" 단순 목록을 대체한다.

원칙: 청사진 픽셀 충실. 색·폰트·weight·간격·패딩·radius는 청사진 Digital Sanctuary.dc.html Frame 1과 tokens.css 값을 그대로 옮긴다. 임의 변형 금지.

범위: 홈 대시보드 화면 구성 + 실데이터 연결(문서 목록·카운트·최근 활동) + placeholder 영역(태그·Starred). 비범위: 폴더 트리 실데이터(S3), 실제 태그 시스템(S6), 즐겨찾기 기능(S3), Graphs/Canvas/Collections(백로그), 탭 중 Recently edited 외 나머지(Starred/Created by me/All pages는 자리만 — 클릭 비활성 또는 후속).

2. 라우팅 — /me → / 흡수

  • 현재: app/page.tsx가 redirect("/me"), /me가 목록.
  • S2: / 자체를 홈 대시보드로 만든다(서버 컴포넌트, getUser 가드 → 미인증 /login).
  • /me는 /로 리다이렉트(기존 북마크·링크 호환). 사이드바 네비의 Home href는 이미 /. New Entry/문서 생성 후 진입 경로 점검.
  • 문서 생성 액션(createDraftDocument)은 그대로 재사용(현재 app/me/actions.ts → app/(home)이나 app/actions.ts로 이동/재노출. 구현 계획에서 경로 확정).

4. 컴포넌트별 픽셀 스펙

4.1 페이지 헤더

  • h1 "Home": font-serif 30px / 600 / -0.02em.
  • 메타: 12.5px / text-ink-muted — "Your knowledge workspace · {N} pages". (contributors 수는 멀티협업 백로그라 생략 — 청사진의 "· 4 contributors"는 넣지 않음.)
  • + New Page 버튼: 우측 정렬, h38, bg-primary text-primary-fg, radius 9, plus(15) 아이콘 + "New Page". 문서 생성 액션 폼.

4.2 탭

  • gap 26, border-b border-border. 항목: Recently edited(활성: 13/600 text-ink + 하단 2px bg-primary 밑줄), Starred·Created by me·All pages(13 text-ink-muted).
  • S2에서는 Recently edited만 동작. 나머지 3개는 자리만(클릭 비활성, 후속). 활성 탭 상태는 정적(라우팅 X) — 토글은 후속.

4.3 JUMP BACK IN — 최근 문서 4개 (실데이터)

  • 라벨 "JUMP BACK IN": 11/600 / .08em / text-ink-faint, margin-bottom 12.
  • 4열 grid(repeat(4,1fr), gap 14). 카드: bg-surface, 1px border-border, radius 10, padding 15×16.
  • 각 카드: 32×32 아이콘 타일(radius 8, margin-bottom 24) + 제목(13/600 text-ink, line 1.3) + 메타(11 text-ink-faint, "{project} · Edited {상대시각}").
  • 아이콘 타일: 청사진은 카드마다 색이 다르나(accent-soft/primary/chip-2), S2는 단순화: 전부 bg-chip-2 text-ink-secondary + file-text(문서 타입/색 분기는 후속). 첫 카드만 강조색(bg-accent-soft text-accent-soft-fg)을 줄지는 구현 시 결정 — 기본은 통일.
  • 데이터: 최근 수정 문서 상위 4건(updated_at desc). 4건 미만이면 있는 만큼. 0건이면 섹션 숨김 또는 빈 카드 1개("첫 문서를 만들어보세요").

4.4 Recently edited 테이블 (실데이터)

  • 섹션 헤더: h2 "Recently edited"(serif 18/600) + 우측 "View all →"(12, arrow-right 13). View all은 후속(비활성 또는 /recent placeholder).
  • 카드: bg-surface, 1px border-border, radius 12, overflow hidden.
  • 헤더 행: bg-surface-alt, border-b border-border, grid 1fr 116px 108px, gap 12, padding 11×18. 컬럼 "NAME / TAG / UPDATED"(10/600 / .07em / text-ink-faint, UPDATED 우측정렬).
  • 데이터 행(grid 동일, padding 13×18, border-b border-border-2, 마지막 행 보더 없음):
    • NAME: file-text(16, text-ink-muted) + 제목(13/500 text-ink, ellipsis) + 브레드크럼(11 text-ink-faint, "Knowledge Base / {project}").
    • TAG: 태그 칩 1개(9.5/600 / .05em / bg-chip text-chip-ink / radius 5 / padding 3×8). 실태그 없음(S6) → project 값을 대문자 칩으로 임시 표시. project 비면 칩 생략.
    • UPDATED: 우측정렬, 상대시각(11.5 text-ink-faint) + 22px 아바타 자리(bg-border-strong 원, 사용자 단일이므로 동일).
  • 데이터: 최근 수정 문서(상위 ~6~10건). 0건이면 빈 상태 행("아직 문서가 없어요. 첫 문서를 만들어보세요.").

4.5 우측 레일 (3 패널, gap 20)

각 패널: bg-surface, 1px border-border, radius 12.

  • Starred(padding 16/16/10): 헤더 star(15, text-star) + "Starred"(13/600). 항목: 즐겨찾기 문서(아이콘+제목 12.5, 높이 34). 즐겨찾기 기능 미구현(S3) → 빈 상태("즐겨찾기한 문서가 여기 표시됩니다"). 레이아웃은 유지.
  • Recent activity(padding 16): 헤더 history(15, text-ink-secondary) + "Recent activity"(13/600). 항목 3건: 좌측 점 불릿(6px, 최신 #3a4a78 / 이전 #c3c8d1) + 본문(12.5/500) + 메타(11 faint). 항목 사이 border-b border-border-2.
    실데이터: 최근 수정 문서 N건을 활동으로 변환 — "Updated \"{title}\" · {상대시각}". (실제 활동 로그 테이블은 백로그.)
  • Quick access(padding 16/16/12): "Quick access"(13/600). 행(높이 36, 12.5): 아이콘 + 라벨 + 우측 chevron-right(faint). 항목: Global Graph(git-fork) · Reading Queue(book-open) · Canvas Mode(pen-tool) · Collections(database). 전부 백로그 → 비활성(aria-disabled, hover 시 "곧 제공", 클릭 X).

4.6 푸터

  • max 940, border-t border-border, 10.5 / .05em / text-ink-faint. 좌: "© {연도} {브랜드}. SCHOLARLY FOCUS MODE ACTIVE." 우: "PRIVACY TERMS API"(링크 placeholder). 브랜드/문구는 placeholder 유지.

7. 테스트

  • vitest 단위:
    • relative-time: 방금/분/시간/일 경계, 미래 방어.
    • RecentlyEditedTable: 행 렌더(제목·브레드크럼·태그칩), 빈 상태(문서 0건 메시지), 컬럼 헤더.
    • JumpBackIn: 최근 4건 렌더, 4건 미만/0건 처리.
    • HomeTabs: Recently edited 활성(aria-current 또는 활성 클래스), 나머지 비활성.
    • 레일: QuickAccess 항목 aria-disabled, Starred 빈 상태, RecentActivity "Updated …" 변환.
  • 회귀: 기존 스위트 + S1 셸 테스트 동반 통과. 깨지면 신규 코드 수정(CLAUDE.md §3).
  • 빌드: / 서버 컴포넌트가 셸 + 대시보드를 정상 컴파일.

8. 미해결 / 검토

  • 탭 토글 동작(Starred/Created by me/All pages)을 S2에서 정적 자리로 둘지, 최소 동작을 줄지 — 기본은 정적 자리.
  • Jump back in 카드 타일 색을 청사진처럼 카드별로 다르게 할지(첫 카드 accent) vs 통일 — 기본 통일, 사용자 취향 시 분기.
  • documents.project를 태그/브레드크럼으로 임시 차용 — S3(폴더)·S6(태그) 도입 시 정식 대체.
  • New Page 생성 후 진입: 바로 /edit/[id](기존 동작) 유지.
